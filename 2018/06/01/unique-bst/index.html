Unique Binary Search Tree

Posted on 2018-06-01

Problem # / Title

96 / Unique Binary Search Tree

Description

Given a positive integer $n$, how many structurally unique BST’s (binary search trees) that store values $1, 2, …, n$ can there be?

Math

Given a sequence $1, 2, …, n$. Define the following terms:

  • $G(n)$: the number of unique BST’s that store a sequence of length $n$.
  • $F(i,n)$: the number of unique BST’s that store a sequence of length $n$, with the number $i$ as the root of the tree

For any tree with number $i$ as the root, its left branch is a BST constructed from the sequence $1,2,…,i_1$, and its right branch is a BST constructed from the sequence $i+1, i+2, …, n$. So its left branch is one of the $G(i-1)$ unique BST’s, and its right branch is one of the $G(n-i)$ unique BST’s.

So $$ F(i,n)=G(i-1)\cdot G(n-i). \tag{1}$$

Also, since the number of unique BST’s for a sequence of length $n$ can be found by adding up the number of BST’s using each number in the sequence as the root, we have
$$G(n) = F(1,n)+F(2,n)+…+F(n,n). \tag{2}$$

Combining equations (1) and (2), we have
$$G(n)=G(0)\cdot G(n-1)+G(1)\cdot G(n-2)+…+G(n-1)\cdot G(0).$$

To compute $G(n)$ for any $n\geq2$, we first need to know what $G(k)$ is for $k=1,2,…,n-1$.

Method

Use bottom-up dynamic programming to tabulate $G(k)$ for $1\leq k \leq n$.

Time complexity

Computing $G(k)$ for each $k = 2,3,…,n$ takes $2k$ lookups in the array, so the total number of lookups is $2(2+3+…+n)=2*\frac{(2+n)(n-1)}{2}=n^2$. So the overall time complexity is $O(n^2)$.
